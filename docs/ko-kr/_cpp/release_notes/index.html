    릴리즈 노트

    2.7.8¶

    Bug Fix¶

    • Anomaly Detection 기능에서 calcScoremap 옵션의 값이 다른 추론 결과에도 영향을 미치던 버그를 해결하였습니다.
    • 핸들파괴 시간이 오래걸리던 문제를 해결하였습니다.

    2.7.7¶

    Bug Fix¶

    • Object Detection 기능에서 Bounding Box의 Center좌표가 표출되지 않던 버그를 해결하였습니다.

    2.7.6¶

    중요 사용방법 변경¶

    • 런타임 환경에서의 기본 추론 옵션이, 모델 파일을 처음 로드할때 유저의 의도와 관계없이 모델 파일에 저장되지 않도록 변경되었습니다. 아래 항목들이 영향을 받습니다. (괄호 안은 기본 값)

      • (공통) calcTime(false), batchSize(1)
      • (CLS) calcClassActivationMap(false)
      • (DET) 해당없음
      • (SEG) calcScoremap(false), calcMask(false), calcObject(true), calcObjectAreaAndApplyThreshold(true), calcObjectScoreAndApplyThreshold(true)
      • (IAD) calcScoremap(false), calcMask(false), calcHeatmap(false), calcObject(true), calcObjectAreaAndApplyThreshold(true), calcObjectScoreAndApplyThreshold(true)
      • (OCR) oversizedImageHandling("do_not_inspect"), maxNumOfTextObjects(0)
      • (IEN) oversizedImageHandling("do_not_enhance")

    • SAIGE VISION Developer 버전 2.0.6 이상에서 학습한 모델 파일이 기존 버전에서는 사용이 불가능합니다. 따라서 런타임의 실행환경인 %SAIGE_HOME%\SaigeVision\module 또한 SAIGE VISION Developer 2.0.6 이상을 요구합니다. (C++ 런타임의 버전과는 관계 없습니다.)

    API 변경¶

    • SaigeModelInfo 항목이 추가되었습니다.
      • targetTextSize
      • targetImageQuality

    2.7.5¶

    Bug Fix¶

    • 일부 모델 파일이 로드되지 않는 버그를 수정하였습니다.

    2.7.4¶

    Bug Fix¶

    • 사용하지 않는 기능에 대한 모듈이 없더라도 정상적으로 동작하도록 수정하였습니다.

    2.7.3¶

    • 내부 모듈을 로드하는 로직을 수정하여 에러 발생을 줄였습니다.
    • 검사 결과 핸들에 지정 또는 생성된 Product Id를 조회하는 API SaigeGetInferenceResultProductId 가 추가되었습니다.
    • V1 호환 API에 MLOps 모니터링을 위한 Product Id를 지정하여 검사하는 API SR_ApplyADTestFrom8bitCharColorArrayWithProductId가 추가되었습니다.

    2.7.2¶

    • 라이센스 체크 방식의 안정성을 개선하였습니다.

    2.7.1¶

    MLOps¶

    • 핸들의 샘플 이미지를 전송하는 로직을 변경하여 Operations > Settings 화면에서 핸들을 조회하면 샘플이미지가 안보이던 경우가 감소합니다.
    • MLOps의 로그에 이미지 데이터가 포함되지 않도록 변경되었습니다.

    API 변경¶

    • SaigeModelInfo의 내용에 학습시 기록된 정보가 다음과 같이 추가되었습니다.
      • description
      • usedGpuName
      • numOfIterations
      • numOfTrainImages
      • numOfValidationImages
      • numOfValidationImages
      • trainingTimeStarted
      • trainingTimeCompleted
      • trainingTimeElapsed

    2.7.0¶

    • 모든 기능 Cls, Det, Seg, IAD, OCR, IEN 을 사용할 수 있으며, MLOps기능이 통합되었습니다.

    Bug Fix¶

    • Debug 모드로 MLOps 로그를 사용하는 경우 메모리 에러를 해결하였습니다.
    • Segmentation 수율 계산을 위한 MLOps 모니터링 정보가 정확하지 않은 문제를 해결하였습니다.

    2.6.1¶

    Bug Fix¶

    • 실행도중 등록한 Inspector의 ID 를 제대로 가져오지 못하던 문제를 해결하였습니다.
    • Inference Handle의 Sample Image가 MLOps 화면에서 제대로 표시되지 않던 문제를 해결하였습니다.

    2.6.0¶

    • MLOps 기능을 사용 가능합니다. (OCR기능은 미지원)
      • SaigeInferFromByteArray, SaigeInferFromByteArrayWithProductId 두 검사 API에서만 사용 가능합니다.
      • 현재 8bit 이미지에서만 사용 가능합니다.

    2.5.3¶

    • Image Enhancement 기능을 사용할 수 있습니다.

    2.5.2¶

    • OCR 기능에서는 최적화 기능의 안정성 문제로 지원을 중단합니다.

    2.5.1¶

    • OCR 기능 업그레이드에 따른 변경사항이 적용되었습니다. 기존에 학습된 모델은 사용이 불가능합니다.
    • OCR 기능의 score값, score threshold값들이 백분율 처리되지 않은 부분 해결하였습니다.

    2.5.0¶

    • OCR 기능을 사용할 수 있습니다.

    2.4.7¶

    • 몇몇 예제 코드에서 type casting 방법이 좀 더 명시적으로 추가되었습니다. (size_t -> int 등)

    중요 API 변경사항¶

    • saige_error.h 헤더 파일이 추가되었습니다. C++ SDK에서 API의 리턴값으로 나올 수 있는 에러코드의 이름을 확인할 수 있습니다.
    • 검사시간을 받는 SaigeGetInferenceResultInspectionTime 의 파라미터가 long (int32)에서 long long (int64)으로 변경되었습니다.
    • SaigeImageType의 enum 변수 이름이 변경되었습니다. 일반적인 이름을 피하기 위함입니다.

    2.4.6¶

    • V1호환 IAD 기능의 결과 구조체인 SR_SimpleSegmentedObject의 형식이 V1 SDK 최종 배포된 버전에 맞춰 업데이트 되었습니다.

    2.4.5¶

    V1 호환 API (시험적, SaigeVisionAPI_V1.h 파일 참조)¶

    • SaigeVision V2의 IAD 기능을 사용하기 위한 V1 호환 API가 제공됩니다.
    • V2의 IAD기능을 사용하기 위한 V1 스타일의 별도 API가 제공됩니다.

    2.4.4¶

    • 일부 기능의 V1 호환 API가 제공됩니다.

    2.4.3¶

    • SaigeModelInfo.classIsNg 항목이 새로 추가되었습니다. 각 클래스별로 학습시 설정한 NG클래스 여부를 나타냅니다.

    2.4.2¶

    • Segmentation의 추론 옵션 변경
      • Mask 계산여부인 calcMask도 Segmentation 추론옵션에 포함됩니다.
      • SegmentedObject 연산 속도를 향상시키기 위해서 calcScoremap, calcMask를 false로 설정할 수 있습니다.
      • 의미있는 추론결과가 없는 현상을 방지하기 위해 calcScoremap, calcMask, calcObject를 모두 false로 설정할 수 없습니다.

    2.4.1¶

    • Object Detection 기능의 내부 구조가 바뀌어 기존에 학습된 모델파일의 직접 사용이 어렵습니다. 새로운 버전에서 학습된 모델을 사용해주세요.

    Bug Fix¶

    • Object Detection 최적화 모델이 일부 이미지 데이터에 대해 런타임 에러가 발생하는 문제를 해결하였습니다.
    • 16bit Channel Depth 이미지의 경우 일부 Util함수에서 시각화가 제대로 표현되지 않던 문제를 해결하였습니다.

    2.4.0¶

    • SaigeInferFromByteArray에 사용되는 이미지 플래그가 다음과 같이 변경되어 더 명확한 의미를 가집니다.
    • SAIGE_IMAGE_ 로 접두어 간소화
    • SAIGE_IMAGE_COLOR 로 RGB 채널 여부를 나타냄
    • SAIGE_IMAGE_16BIT 로 채널당 픽셀 데이터의 depth를 나타냄. (8bit unsigned int 또는 16bit unsigned short)
    • SAIGE_IMAGE_ALPHA 로 Alpha 채널의 유무를 나타냄
    • SAIGE_IMAGE_BGR 로 컬러 이미지의 채널 순서를 나타냄 (기본: RGB)
    • 이하 매크로들을 통해서 위의 네 플래그의 조합을 제공합니다. 매크로 이름을 통해 이미지 배열 형식을 명확하게 표현합니다.
      • SAIGE_IMAGE_8BIT_GRAY
      • SAIGE_IMAGE_24BIT_RGB
      • SAIGE_IMAGE_24BIT_BGR
      • SAIGE_IMAGE_32BIT_RGBA
      • SAIGE_IMAGE_24BIT_BGRA
      • SAIGE_IMAGE_16BIT_GRAY
      • SAIGE_IMAGE_48BIT_RGB
      • SAIGE_IMAGE_48BIT_BGR
      • SAIGE_IMAGE_64BIT_RGBA
      • SAIGE_IMAGE_64BIG_BGRA
    • 위 이미지 플래그 변경사항과 관련하여 제공된 모든 SaigeVisionUtils.h의 함수들이 변경되었습니다. Infer API와 다르게 항상 RGB로 작동합니다.
    • 이미지 데이터를 받는 모든 함수에서 자료형이 const void*로 변경되었습니다. 8bit unsigned char (uint8) 또는 16bit unsigned short (uint16)을 따로 구분하지 않기 위함입니다.
      • uint8, uint16, float32, double64를 구분하기 위한 enum Type이 추가되었습니다. 일반적인 상황에서 Float 타입은 사용되지 않으며, 히트맵 배열과 같이 일부 예시나 보조 함수에서 사용될 수 있습니다.
    • SaigeGetImageData: 이미지 데이터의 채널 수와 자료형을 받아옵니다. 또한, 원본 채널과 데이터 타입을 보존합니다.

    • 원본 이미지에 검출된 결과를 시각화하는 함수들도 channels와 더불어 type 정보 (8bit, 16bit)를 받게 됩니다. 자세한 내용은 예제 코드의 변경된 부분을 참고합니다.

      • SaigeDrawDetectedObject
      • SaigeDrawSegmentedObject
      • SaigeDrawImage
      • SaigeDrawImageOverBase

    • 빌드된 DLL 파일에 버전 정보를 추가하였습니다.

    Bug Fix¶

    • 일부 이미지 데이터에 대해 Object Detection 기능의 최적화가 적용되지 않던 버그가 해결되었습니다.
    • calcScoremap 추론옵션이 제대로 저장되지 않는 문제 해결되었습니다.
    • 추론 최적화 임시 파일 이름에 기존 기능 확장자(.saigecls 등)를 포함시켜 이름중복 문제를 해결하였습니다.
    • 모델파일에 저장된 값을 불러올 때 예외처리를 통해 적절한 에러코드를 반환하도록 변경되었습니다.
    • Object Detection 기능의 모델정보 및 추론옵션과 관련하여 classIndex: 0이 "OK" 클래스로 예약되어있어야 하는 동작이 제대로 적용되지 않았습니다. 아래와 같이 각 항목별로 수정하였습니다.
      • SaigeModelInfo.classNames, classColors에 적절한 값을 보여줍니다. 만약 "OK" 클래스가 없다면 해당 모델은 옳바르지 않은 Developer 버전에서 학습된 모델입니다.
      • 따라서, 검출된 SaigeDetectedObject의 classIndex값은 항상 1 이상입니다.
      • SaigeInferenceOption.objectAreaThresholdPerClass, objectScoreThresholdPerClass, maxNumOfDetectedObjects에서 "OK" 클래스에 해당하는 0번째 값은 기본값 0으로 고정되어있으며 조작하여도 Set Option API에서 반영되지 않습니다.

    Bug Fix¶

    2.3.2¶

    Bug Fix¶

    2.3.1¶

    • 핸들 생성 시 에러 메세지 처리가 좀 더 구체화 되었습니다. (예: 경로에 없는 파일인 경우)

      • SaigeOptimizeInferenceHandle API 가 삭제되었습니다.
      • SaigeCreateInferenceHandle 호출 시 flag를 통해 optimize 여부를 결정합니다.
      • 최적화 level 옵션이 삭제되었습니다.
    • oversizedImageHandling 옵션들의 매크로 이름이 변경되었습니다.
    • 기존 Object Detection 기능을 위해 존재하던 변수, 상수, 매크로들의 이름이 변경되었습니다.

    2.2.0¶

    • Classification 기능을 사용할 수 있습니다.

    2.1.2¶

    Bug Fix¶

    • Project Type를 잘못 사용하는 경우 (e.g. IAD 핸들 - SEG 모델) 발생하는 모델파일 손상 문제를 해결하였습니다.

    2.1.1¶
